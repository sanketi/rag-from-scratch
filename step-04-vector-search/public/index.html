RAG from Scratch / Step 04

Vector Search with ChromaDB

Chunk a document, embed it with CIS, store it in a vector database, then find the best matches for any question.

Learn it in 5 steps

  1. Steps 02–03 recap: you turned text into numbers (embeddings) and cut documents into chunks.
  2. Vector DB = smart filing cabinet. Each chunk's embedding goes in a drawer. ChromaDB keeps millions of drawers and finds the closest ones fast.
  3. Index time: chunk → embed each chunk → store in Chroma. You do this once when a document arrives.
  4. Query time: embed the user's question → ask Chroma "which drawers are closest?" → get back the top chunks.
  5. Step 03 used keyword matching. This step uses meaning — "PTO days" matches "paid time off" even if the words differ.
